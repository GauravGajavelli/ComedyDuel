# Physical Performance Observations

**Merged into `working/laugh_points.md`.**

Physical performance data is now captured as annotation layers on the
laugh-point partition segments (physical_state, physical_change,
physical_load fields). This avoids maintaining two competing segmentations
and ensures physical observations are anchored to the same text boundaries
as laugh points and structural events.

See `working/laugh_points.md` for the format and collection instructions.
